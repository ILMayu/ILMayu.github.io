面向切面编程AOP

spring中基于XML的AOP配置步骤

1、把通知Bean也交给spring来管理

2、使用aop:config标签表明开始AOP的配置

3、使用aop:aspect标签表明配置切面

    id属性:是给切面提供一个唯一 标识ref属性: 是指定通知类bean的Id。

4、在aop:aspect标签的内部使用对应标签来配置通知的类型

四种常用通知类型

aop:before        前置通知:在切入点方法执行之前执行

aop:after-returning  后置通知:在切入点方法正常执行之后值。它和异常通知永远只能执行一个。

aop: after- throwing 异常通知:在切入点方法执行产生异常之后执行。它和后置通知永远只能执行一个

aop: after       配置最终通知:无论切入点方法是否正常执行它都会在其后面执行

标签属性

method属性:用于指定Logger类中哪个方法是前置通知

pointcut属性:用于指定切入点表达式，该表达式的含义指的是对业务层中哪些方法增强

pointcut-ref属性:引用aop:pointcut配置的切入点表达式

标准的表达式写法

例子：public void com.lxl.service.impl.AccountServiceImpl.saveAccount()

1. 访问修饰符可以不写,返回值可以使用通配符，表示任意返回值

2. 包名可以使用通配符，表示任意包。但是有几级包，就需要写几个*.

    例子：* *..AccountServiceImpl.saveAccount()

3. 类名和方法名都可以使用*来实现通配

    例子：* *..*.*()

参数列表

1. 可以直接写数据类型

2.基本类型直接写名称  int

3.引用类型写包名.类名的方式  java.lang.String

全通配写法

​

package com.lxl.service;

public interface IAccountService {
void saveAccount();
void updateAccount(int i);
int deleteAccount();
}

package com.lxl.service.impl;

import com.lxl.service.IAccountService;

public class AccountServiceImpl implements IAccountService {

@Override
public void saveAccount() {
    System.out.println("保存");
}

@Override
public void updateAccount(int i) {
    System.out.println("更新" + i);
}

@Override
public int deleteAccount() {
    System.out.println("删除");
    return 1;
}

}

package com.lxl.utils;

import org.aspectj.lang.ProceedingJoinPoint;

public class Logger {
public void beginLog(){
System.out.println(“aop:before 日志”);
}

public Object aroundLog(ProceedingJoinPoint pjp){
    Object rtValue = null;
    try {
        Object[] args = pjp.getArgs();
        System.out.println("aop:around 的before");
        rtValue = pjp.proceed(args);
        System.out.println("aop:around 的after-returning");
        return rtValue;

    } catch (Throwable t) {
        //Exception类无法捕获此类异常
        System.out.println("aop:around 的after-throwing");
        throw new RuntimeException(t);
    } finally {
        System.out.println("aop:around 的after");
    }

}

public void returningLog(){
    System.out.println("aop:after-returning 日志");
}

public void throwingLog(){
    System.out.println("aop:after-throwing 日志");
}

public void afterLog(){
    System.out.println("aop:after 日志");
}

}

<bean id="accountService" class="com.lxl.service.impl.AccountServiceImpl"/>
<bean id="logger" class="com.lxl.utils.Logger"/>

<aop:config>

        <!--切入点配置-->
    <aop:pointcut id="pt1" expression="execution(public * com.lxl.service.impl.AccountServiceImpl.*(..))"/>
    <aop:aspect id="LogAdvice" ref="logger">
        <!--前置通知-->
        <aop:before method="beginLog" pointcut-ref="pt1"/>
        <!--环绕通知-->
        <aop:around method="aroundLog" pointcut-ref="pt1"/>
        <!--后置通知-->
        <aop:after-returning method="returningLog" pointcut-ref="pt1"/>
        <!--异常通知-->
        <aop:after-throwing method="throwingLog" pointcut-ref="pt1"/>
        <!--最终通知-->
        <aop:after method="afterLog" pointcut-ref="pt1"/>
    </aop:aspect>
</aop:config>
